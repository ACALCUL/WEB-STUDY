  • HTML
    • HyperText 란?

      HyperLink를 통해 독자가 한 문서에서 다른 문서로 즉시 접근할 수 있는 텍스

    • 마크업 언어란?

      문서가 화면에 표시되는 형식을 나타냄+데이터의 논리적인 구조를 명시하기 위한 규칙을 정의

      프로그래밍 언어라고 부르기에는 부족→조건문, 메모리 접근이 없으며(튜링 완전하지 않으며), 단순 문서 기록용에 가깝기 때문

      마크업 언어의 예시로는 HTML, 마크다운 등이 있다.

    • 브라우저는 어떻게 HTML을 읽고 화면에 표시할까?
      1. 사용자가 특정 동작을 한다(사용자 인터페이스)
      1. 브라우저가 웹 페이지를 요청한다.(브라우저 엔진)
      1. 서버는 HTML 문서를 보낸다.
      1. 렌더링 엔진이 다음을 진행한다.

      ① 렌더링 엔진은 HTML 문서를 파싱 하여 DOM 트리를 구축한다.

      ② 그다음 외부 CSS 파일과 함께 포함된  스타일 요소를 파싱한다.

      ③ DOM 트리와 ②의 결과물을 합쳐 렌더 트리를 구축한다.

      ④ 렌더 트리 각 노드에 대해 화면 상에서 배치할 곳을 결정한다.

      ⑤ UI 백엔드에서 렌더 트리의 각 노드를 그린다.

      1. 결과적으로 최종 렌더링된 웹페이지가 표시된다.
    • HTML의 기본 구조
      • 문서 형식 선언

        문서의 내용이 시작되기 전 어떤 마크업 언어 형식으로 작성되었는지 명시

        ex. <!DOCTYPE html>: html5를 형식으로

      • <html>

        실제 문서의 내용을 표시하는 태그, <html>안에 <head>나 <body>가 들어간다.

        ex. <html langs=”ko”> … </html>

      • <head>

        브라우저에 문서의 정보를 전달하는 역할

        제목, 스타일, meta 등이 들어감

      • <body>

        화면에 표시할 콘텐츠를 입력

        body 안에 텍스트나 이미지, 미디어 요소 등의 다양한 콘텐츠를 포함하여 웹페이지를 풍성하게 꾸밀 수 있다.

    • HTML의 버전들

      1990년대에 최초의 HTML이 만들어졌다. 그리고 꾸준히 발전을 거듭하여 현재 쓰고 있는 HTML5 이 2008년 발표되어 현재까지 쓰이고 있다. HTML5는 비디오, 오디오 등의 콘텐츠를 ActiveX없이 브라우저에서 쉽게 볼 수 있는 것을 목적으로 한다. 그리고 HTML5에서 마이너 업데이트를 걸쳐 2017년 말에는 HTML5.2 표준도 확정한 상태이다. 현재도 HTML은 발전되고 있다.

    • 선언문? <!DOCTYPE html>, <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd"> 등

      문서 형식 선언(DOCTYPE)은 문서가 어떤 마크업 언어 형식으로 작성되었는지 명시하는 역할을 한다.

      HTML5의 표기법은 <!DOCTYPE html>로 매우 단순하지만

      구버전 HTML의 표기법은 <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">와 같이 복잡하게 써야 한다.

    • 헤더에 들어가는 내용들
      • <meta>, <title>, <link>, <style>, <script>, ...
      • <meta>

        해당 문서에 대한 정보(metadata)를 정의할 때 사용

        인코딩 방식, 검색 엔진을 위한 키워드, 웹 페이지에 대한 설명, 문서의 저자 등을 정의할 수 있다.

        ex) <meta charset="UTF-8"> <meta name="keyword" content="HTML, meta, tag, element, reference"> <meta name="viewport" content="width=device-width, initial-scale=1.0">

      • <title>

        문서의 제목을 정의할 때 사용

        브라우저의 제목으로 사용되며, 검색 결과 페이지에서도 해당 페이지의 제목으로 나타남

        모든 HTML문서에는 <title> 요소(element)가 필요하지만, 두 개 이상은 불가능하다

        ex) <title>HTML 문서의 제목입니다.</title>

      • <link>

        해당 문서와 외부 소스 사이의 관계를 정의할 때 사용

        주로 외부 스타일 시트를 연결할 때 사용

        ex) <link rel="stylesheet" type="text/css" href="/examples/media/expand_style.css">

        (rel: relation, href: hypertext reference)

      • <style>

        HTML 문서의 스타일 정보를 정의할 때 사용

        <style> 요소는 해당 요소가 포함된 HTML 문서의 콘텐츠에 적용되는 CSS를 명시하고 있으며, 여러 개의 <style> 요소를 포함할 수 있다.

        별도의 파일로 저장된 외부 스타일 시트는 <link> 요소를 사용하여 참조할 수 있다.

        ex)

        <style>

        body

        { background-color: lightpink }

        h1

        { color: red; }

        p

        { text-decoration: underline; }     </style>

      • <script>

        자바스크립트와 같은 client-side scripts를 정의할 때 사용

        (cilent-side script: 웹에서 사용되는 스크립트 언어 중 클라이언트 사이드에서 실행되는 스크립트 언어)

        ex)

        <script type="application/javascript"> document.getElementById("para").innerHTML = "자바스크립트가 실행되었습니다.!"; </script>

    • body에 들어가는 내용들
      • <h1>, <p>, <div>, <span>, <ul>, <ol>, <li>, <table>, <tr>, <td>, <form>, <input>, <button>, ...
      • <h1>~<h6>

        제목을 보여주는 태그

        h1에서 h6로 갈수록 글자크기가 작아진다.

      • <p>

        <p>태그 안에 들어가 있는 요소들은 하나의 문단을 나타낸다.

      • <div>

        CSS로 꾸미기 전까지는 어떤 영향도 주지 않지만, 나중에 class나 id속성으로 꾸미기 쉽도록 돕는 등의 용도로 사용한다.

        ex) <p>이 문단은 블록 레벨 요소입니다. 부모 요소와 구분할 수 있도록 배경 색을 입혔습니다.</p>

      • <span>

        <div>와 같이 구문 콘텐츠를 위한 컨데이너 이지만, 블록 레벨 요소인 <div>와 달리 <span>은 인라인 요소이다.

        ex) <p>Add the <span class="ingredient">pine nuts</span> and <span class="ingredient">garlic</span> to a blender and blend into a paste.</p>

      • <ul>

        <ul> 태그는 순서가 없는 HTML 리스트(list)를 정의할 때 사용한다.

        주로 <li>와 함께 표현되며, <ul>은 하위요소를 묶어줄 때 필요하다.

        <ul>로 묶은 뒤에 <li>로 요소를 감싸면 각 글자 앞에 불릿이 생긴다.

        ex)

        <ul> <li>아메리카노</li> <li>카페라떼</li> <li>핫초코</li> <li>차 <ul> <li>녹차</li> <li>홍차</li> <li>자스민차</li> </ul> </li>

        </ul>

      • <ol>

        <ol> 태그는 순서가 있는 HTML 리스트(list)를 정의할 때 사용한다.

        주로 <li>와 함께 표현되며, <ol>은 하위요소를 묶어줄 때 필요하다.

        <ol>로 묶은 뒤에 <li>로 요소를 감싸면 각 글자 앞에 숫자가 생긴다.

        사용법은 <ul>과 같다.

      • <li>

        <li> 태그는 HTML 리스트(list)에 포함되는 아이템(item)을 정의할 때 사용한다.

        리스트에 포함 시킬 아이템을 각각 <li>태그로 감싸면 된다.

      • <table>

        <table> 태그는 데이터를 포함하는 셀(cell)들의 행과 열로 구성된 2차원 테이블을 정의할 때 사용

        ex)

        <table> <tr> <th>밥류</th> <th>면류</th> <th>분식류</th> </tr> <tr> <td>김밥</td> <td>라면</td> <td>떡볶이</td>

        </tr> </table>

      • <tr>

        table row를 의미하며 <tr>로 감싼요소는 각 행을 이룬다.

      • <td>

        table data cell을 의미하며, <tr> 안에 <td>로 감싼 요소는 각 열을 이룬다.

      • <th>

        table header cell을 의미하며, 기본적인 사용법은 <td>와 비슷하지만 헤더 정보를 저장하는 용도로 사용되므로 굵은 폰트로 중앙 정렬된다.

      • <form>

        <form> 태그는 사용자로부터 입력을 받을 수 있는 HTLM 입력 폼(form)을 정의할 때 사용

      • <input>

        <input> 태그는 사용자로부터 입력을 받을 수 있는 입력 필드(input filed)를 정의할 때 사용

      • <button>

        클릭 가능한 버튼을 나타냄

        ex)

        <button class="favorite styled" type="button"> Add to favorites </button>

      추가 부분

      • <!- - - ->

        주석을 나타낸다.

      • <br>

        break를 의미하며,

        줄바꿈 요소로 따로 닫는 태그와 여는 태그가 필요하지 않고 문장 끝에 <br>을 붙인다.

      • <a>

        anchor를 의미하며, <a> 요소는 href 특성을 통해 다른 페이지나 같은 페이지의 어느 위치, 파일, 이메일 주소와 그 외 다른 URL로 연결할 수 있는 하이퍼링크를 만듦

        ex)

        <a href="https://example.com”>Website</a>

      • <img>

        <img> 태그는 HTML 문서에서 이미지(image)를 정의할 때 사용

        <img> 요소에는 src 속성과 alt 속성을 반드시 명시해야만 함

        src(source)는 가져올 곳, alt는 이미지가 없을 때 보여줄 대체 텍스트를 표시

        ex)

        <img src="/examples/images/img_monalisa.png" alt="모나리자">

      • <label>

        <label> 태그는 사용자 인터페이스(UI) 요소의 라벨(label)을 정의할 때 사용

        for 속성을 사용하여 다른 요소와 결합할 수 있으며. for속성에 결합하고자 하는 요소의 id 속성값을 넣음

        ex)

        <input type="radio" name="ages" id="teen" value="teenage"> <label for="teen">10대</label><br>

    • HTML5의 새로운 태그들: semantic tag
      • <header>, <footer>, <nav>, <article>, <section>, <aside>, ...
      • <header>

        <header> 태그는 문서나 특정 섹션(section)의 헤더(header)를 정의할 때 사용

        헤더에는 보통 도입부에 해당하는 콘텐츠나 네비게이션 링크의 집합 등과 같은 정보를 포함

        ex)

        <header>

        <h3>날씨 정보</h4> <h4>2월 19일</h4> <p>- 기상청 제공 -</p> </header> <p>서울 : 맑음</p>

      • <footer>

        <footer> 태그는 문서나 특정 섹션(section)의 푸터(footer)를 정의할 때 사용

        푸터에는 저자 정보, 저작권 정보, 연락처 등과 같은 정보를 포함

      • <nav>

        <nav> 태그는 다른 페이지 또는 현재 페이지의 다른 부분과 연결되는 네비게이션 링크(navigation links)들의 집합을 정의할 때 사용

      • <article>

        <article> 태그는 해당 문서나 페이지 또는 사이트와는 완전히 독립적으로 구성할 수 있는 요소를 정의할 때 사용

      • <section>

        <section> 태그는 HTML 문서에 포함된 독립적인 섹션(section)을 정의할 때 사용

      • <aside>

        <aside> 태그는 페이지의 다른 콘텐츠들과 약간의 연관성을 가지고 있지만, 해당 콘텐츠들로부터 분리시킬 수 있는 콘텐츠로 구성된 페이지 영역을 정의할 때 사용

    • 검색엔진에서 내 사이트를 잘 찾게 하려면 어떻게 해야할까?
      1. 각 검색엔진의 웹마스터 도구에 들어가 내 사이트임을 인증한다.
      1. robots.txt를 만들어 어느 정보까지 접근할지를 정한다.
      1. 검색엔진 크롤러가 내 사이트를 찾기 쉽게 하기 위해서 sitemap.xml을 만든다.
      1. 그 다음 검색 엔진 최적화(SEO, Search Engine Optimization)를 통해 더욱 검색이 쉽게 되도록 만든다.
    • 웹 개발자 도구에서 어떤 내용을 볼 수 있을까?
      1. 요소(Elements): 웹페이지의 HTML 구조, CSS 스타일 및 레이아웃을 보거나 조작할 수 있다.
      1. 콘솔(Console): 콘솔 패널은 JavaScript 코드를 로깅(logging)하고 디버깅하는데 사용한다. 이때 스크립트에서 생성된 오류 메시지, 경고 및 로그를 표시한다.
      1. 네트워크(Network): 웹페이지에서 생성된 네트워크 요청을 볼 수 있다.
      1. 소스(Source): JavaScript 코드를 디버깅하고 분석한다.
      1. 성능(Performance): 웹 페이지 또는 웹 애플리케이션의 성능을 분석할 수 있다.
      1. 애플리케이션(Application): 쿠키, 로컬 스토리지 및 세션 스토리지를 포함하여 웹 스토리지를 관리하고 분석하기 위한 도구를 제공
      1. 보안(Security): 웹 페이지의 보안 측면에 대한 정보를 제공
      1. 접근성(Acessibility): 웹페이지의 접근성 문제를 확인

      그 외에도 웹 개발자 도구에는 다양한 패널들이 있다.

    • 모바일 웹 페이지는 어떻게 만들까?

      반응형 디자인을 사용하여 화면 크기와 비율에 따라 레이아웃과 스타일이 바뀌는 코드를 넣어야 한다.

      그리고 모바일은 pc환경과 달리 터치가 주 입력장치이므로 이를 고려하여 모바일 친화적인 레이아웃과 디자인으로 사이트를 꾸며야 한다.

  • CSS(Cascading Style Sheets)
    • CSS 없이는 웹 페이지를 만들 수 없을까?

      CSS없이 html만으로도 웹 페이지를 만들 수 있다.

      하지만, CSS를 사용하지 않고 웹 사이트를 만드는것을 권장하지 않는다.

      • 가능하다면 왜 우리는 CSS를 사용할까?

        우리가 CSS를 쓰는 이유는 디자인적 요소 때문이다.

        CSS는 웹 페이지 스타일링에 유연성, 제어 및 일광성을 제공한다.

        따라서 CSS를 통해 반응형 레이아웃을 만들고, 사용자 지정 글꼴을 적용하고, 매력적인 요소를 디자인하고, 다양한 장치 및 크기에 맞게 표현할 수 있다.

    • CSS는 어떻게 HTML에 적용할까?
      • inline style

        style속성을 사용하여 요소에 직접적으로 사용하는 CSS 스타일이다

        ex)

        <p style="color: red;">This is a red paragraph.</p>
      • internal style

        head안에 <style>태그를 넣어 html 문서에서 스타일을 정의하는 CSS 스타일이다.

        ex)

        <!DOCTYPE html>
<html>
<head>
	<style>
		p {
			color: blue;
			font-size: 16px;
		}
	</style>
</head>
<body>
	<p>This is a blue paragraph with font size 16px.</p>
</body>
</html>

      • external style

        별도의 css파일을 만들고 head안에 <link>태그를 통해 HTML과 CSS문서를 연결해주는 CSS 스타일이다.

        ex) 

        <!DOCTYPE html>
<html>
<head>
	<link rel="stylesheet" type="text/css" href="styles.css">
</head>
<body>
	<p>This paragraph will be styled using the rules from the external CSS file.</p>
</body>
</html>
    • CSS의 기본 구조

      h1{color:red;

      font-size:12px;}

      에서 h1은 선택자(selection), color와 font-size는 속성(property) 그리고 red와 12px은 속성 값(value)이다.

    • CSS의 버전들

      1996년 CSS1이 발표되어, 현재는 CSS3가 최신 버전이다.

      정기적으로 새로운 모듈과 업데이트를 구현하고 있으며, 이 때문에 HTML과 같이 지속적으로 새로운 버전이 나오고 있다.

    • 선택자
      • 전체 선택자

        전체 요소에 영향을 주는 선택자

        선택자에 *를 적는다.

        ex) *{color: red;}

      • 태그 선택자

        특정 태그에 영향을 주는 선택자

        선택자에 태그를 적는다.

        ex) p{color: red;}

      • 클래스 선택자

        특정 클래스 요소에 영향을 주는 선택자

        선택자에 .과 클래스이름을 적는다.

        ex) .class1{color: red;}

      • ID 선택자

        특정 아이디를 가진 요소에 영향을 주는 선택자

        선택자에 #과 아이디 이름을 적는다

        ex) #id1{ color: red;}

      • 하위 선택자

        특정 요소의 하위에 있는 특정 요소에 영향을 주는 선택자

        선택자에 특정 요소를 먼저 적고 띄운 다음 하위 요소를 적는다.

        띄어쓰기를 여러번 사용하여 여러 3개 이상의 선택자를 넣을 수 있다.

        ex) div ul{color: red;}

      • 자식 선택자

        하위 선택자와 비슷하지만, 자식 선택자는 바로 밑의 자식의 특정 요소에만 영향을 주며, 자식의 자식과 같이 모든 하위에 있는 특정 요소에 영향을 주지는 않는다.

        띄어쓰기 대신 >를 사용한다.

        ex div > ul{color: red;}

    • 상속

      부모(상위)요소에 적용된 값을 자식(하위)요소가 물려받는 것

      하지만, 태그와 속성에 따라 상속이 되지 않는 경우가 있다.

      이런 경우에는 inherit를 사용하면 된다.

      ex)

      <!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width">
  <title>JS Bin</title>
  <style>
    h1 {
      color : orange
    }
    a {
      color : inherit;

    }
  </style>
</head>
<body>
  <h1>
    <p>p태그는 자동으로 상속됩니다.</p>
    <a href="#">하지만 a태그는 inherit을 이요해야 상속됩니다.</a>
  </h1>
</body>
</html>
    • 우선순위

      CSS의 적용 우선 순위는 아래와 같다.

      1. 속성 값 뒤에 !important를 붙인 경우(권장은 하징 않음)
      1. 인라인 스타일로 지정한 속
      1. ID 선택자로 지정한 속성
      1. 클래스 선택자 및 속성 선택자
      1. 태그 이름으로 지정한 속성
      1. 상속된 속성 및 범용 선택자(*)
    • 단위
      • 절대 단위

        사용자 환경이 변해도 크기가 고정되는 크기의 단위를 말한다.

        대표적으로 px가 있다

      • 상대 단위

        사용자 환경에 따라 크기가 변하는 크기의 단위를 말한다.

        상위 요소를 기준으로 크기를 조정하는 em, 루트 요소인 <html>을 기준으로 크기를 조정하는 rem, 뷰포트(보여지는 화면의 크기) 기준 너비인 vw, 높이인 vh, 그리고 % 등이 있다.

    • 색상을 표현하는 방법
      1. 색상 이름을 적기

        ex) color: red; color: grey;

      1. RGB 색상값으로 표현

        순서대로 R, G, B의 양을 정하면 되며, 0~255까지 지정이 가능하다.

        (색상 하나당 1byte이므로)

        ex) color: rgb(20, 50, 255)

      1. 16진수 색상값으로 표현

        각 2자리마다 순서대로 R, G, B의 양을 정하면 되며, 00~FF까지 지정 가능하다.

        똑같이 색상 하나당 1byte이므로 16진수 2자리가 한 색상을 나타낸다

        ex) color: #0000FF;

    • 박스 모델

      HTML 요소는 박스 모양으로 구성되며, 이 형태를 박스 모델이라고 부른다.

      Content를 감싸는 패딩(padding), 그 바깥에 border 마지막 여백인 margin이 순서대로 있다.

      .box {
  color: white;
  background-color: green;
  width: 200px;
  height: 30px;
  padding: 20px;
  border: 5px;
  margin: 15px;
}

      위와 같이 각 테두리의 두께를 설정할 수 있다.

    • display 종류들
      • display: none;

        화면에서 사라진다.

      • display: block;

        기본적으로 부모 컨테이너의 전체 너비를 차지하며 앞뒤에 새 줄을 만든다.

        따라서 세로로 쌓이고 너비와 높이를 설정 가능

        블록 요소는 다른 블록 및 인라인 요소를 포함 가능

        <div>, <p>, <h1>에서 block이 디폴트

      • display: inline;

        앞이나 뒤에 줄 바꿈을 만들지 않는다.

        콘텐츠에 따라 필요한 공간만 차지하며 너비와 높이를 설정 불가

        인라인 요소는 다른 인라인 요소를 포함할 수 있지만 블록 요소는 포함 불가

        <span>, <a>, <strong>에서 inline이 디폴트

        width, height, padding, margin의 속성 값이 모두 무시된다.

      • display: inline-block;

        인라인 요소처럼 작동하지만 너비, 높이, 패딩 및 여백을 설정할 수 있다.

        줄 바꿈을 만들지 않으며 같은 줄에서 서로 옆에 앉을 수 있음

        <input>, <button>, <img>에서 inline-block이 디폴트

      • display: flex;

        유연하고 반응이 빠른 레이아웃을 허용하는 플렉스 컨테이너를 만든다.

        Flexbox는 컨테이너 내의 하위 요소에 대한 강력한 정렬 및 배포 기능을 제공

        수직 또는 수평 중앙 정렬, 균등 간격 및 요소 재정렬이 용이

        Flexbox는 반응형 및 동적 레이아웃을 만드는 데 사용

      • display: grid;

        복잡한 그리드 기반 레이아웃을 생성할 수 있는 그리드 컨테이너를 생성

        Grid는 행과 열 모두에 요소를 배치할 수 있는 2차원 레이아웃 시스템을 제공

        그리드 내에서 요소의 배치 및 정렬을 정밀하게 제어할 수 있으므로 고급 반응형 디자인을 만드는 데 이상적

    • CSS에서 변수를 사용할 수는 없을까?

      사용가능하다.

      :root {
  --main-bg-color: brown;
}

      위의 코드와 같이 선언하며,

      element {
  background-color: var(--main-bg-color);
}

      아래의 코드와 같이 var()함수를 통해 적용할 수 있다.

    • CSS로 움직이는 효과를 줄 수 있을까?

      CSS는 지정된 기간 동안 요소의 다양한 속성에 애니메이션을 적용할 수 있는 CSS 애니메이션이라는 강력한 메커니즘을 제공

      JavaScript 또는 외부 라이브러리 없이도 웹 페이지의 요소에 이동 및 전환을 추가하는 방법을 제공

    • CSS로 애니메이션을 만들 수 있을까?

      CSS로 애니메이션을 만들려면 @keyframes와 animation 속성을 사용@keyframes는 애니메이션의 키프레임 또는 중간 상태를 정의하는 반면 animation 속성은 애니메이션을 요소에 적용합니다.

    • 화면 크기에 따라 다른 스타일을 적용하고 싶을 때는 어떻게 해야할까?

      @media rule을 통해 특정 화면 픽셀에 도달했을 때의 스타일을 지정할 수 있다.

  • CSS3와 HTML5는 튜링 완전(Turing complete)한가?

    튜링완전이란 튜링 기계로 설명할 수 있는 모든 계산을 수행할 수 있는 언어의 특성을 말한다.

    튜링완전하려면 조건부 분기, 루프 또는 재귀, 가변 스토리지 기능이 있어야 한다.

    그리고 두 언어를 함께 쓰면 튜링완전하다고 증명된 세포자동차를 만들 수 있기 때문에 튜링 완전하다.
